§ 02 · WHY CODE-FIRST FITS BUSINESS AI

Business AI is already a code-first team.

Data scientists already write Python every day. Their entire toolchain — modelling, experiments, features, pipelines — is code-first. An agentic SDK is one more import, not a new paradigm.

The stack the team already uses

Modelling: pandas, scikit-learn, statsmodels, PyTorch, JAX. Notebook-first, then packaged.

Lifecycle: MLflow / Weights & Biases for experiments, Feast for features, Airflow / Dagster / Kubeflow for pipelines, FastAPI for serving. All code-first.

Cloud: Python SDKs from every cloud provider, every vector DB, every observability tool. A code-first agentic SDK fits this stack natively.

What an agent is, technically

An agent is not a workflow. A workflow is a static DAG. An agent decides at runtime what to do next based on tokens it just emitted.

That decision logic — planning, tool selection, memory compaction, retry, refusal — is conditions on state, not edges on a canvas. It belongs in code.

This is why the OSS consensus — LangGraph, MS Agent Framework, Google ADK — is uniformly code-first.

The fit is direct. A typed Python library compounds with everything the Business AI team already knows: testing in pytest, packaging in pyproject.toml, shipping in containers, observing in OpenTelemetry. Nothing new to learn that isn't agent-specific.

§ 03 · CONCRETELY

Eight lines. Every enterprise concern, wired.

A registered, schema-validated, audit-emitting, policy-checked, budget-tracked tool — with input/output models, version pinning, and JSON-Schema export — is one decorator away.

@tool(name="order_lookup", version=2)
async def order_lookup(payload: OrderLookupIn) -> OrderLookupOut:
    # Pydantic-validated I/O. Span + audit + budget + policy fire automatically.
    order = await orders_service.get(payload.order_id, tenant_id=payload.tenant_id)
    return OrderLookupOut.model_validate(order)
Typed tool contracts & schema versioning
Pydantic models keyed by (name, version); JSON Schema export for cross-language clients. pydantic.dev
Deterministic tests in CI without a live cluster
DI-injected fakes for LLM, OPA, MCP, storage. pytest + golden replays. No vendor runtime required.
OpenTelemetry traces & metrics by default
Every LLM call, tool call, and policy decision emits a span. Joins the bank's existing OTel stack. OTel GenAI semconv
Policy-as-code via OPA / Rego
Authorization is a versioned, reviewable artefact — not a vendor checkbox. Fail-closed by default. openpolicyagent.org
Per-tenant budget & cost attribution
Token + USD limits enforced before every LLM call; usage recorded after. Refund hooks on failure.
Audit trail with PII redaction
Every prompt, tool call, and policy decision audited; configurable redaction profile for regulated payloads.
Vendor portability via ABCs
Swap LiteLLM → Bedrock, OPA → Cedar, Postgres → DynamoDB by changing one DI binding. No agent rewrites.
Reviewable in PRs · reproducible from a SHA
Every change is a commit; every commit has an author, reviewer, CI run. Regulators and auditors already speak this language.
