YOUR CHARTER

You make it impossible for an agent team
to ship something unsafe by accident.

You own the cross-cutting concerns — auth, audit, observability, budgets, retries, schemas, secrets, MCP, memory, the test seam. The business AI team owns prompts, tools, schemas, evaluations, and business outcomes. The SDK is the contract between you.

If a regulated agent ships and it can't be audited, that's on you. If it can be audited but the prompts give bad answers, that's on the business AI team. Keep the line clean.

CH 01

Day 1 setup.

If these steps complete cleanly, your environment is ready. If any step fails, that's your first ticket.

# 1. clone
git clone git@github.com:eaap/ai-core-sdk.git
cd ai-core-sdk

# 2. virtualenv + dev deps
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev,sentry,datadog,storage-aws]"

# 3. run the test matrix — should be green on a clean checkout
pytest -q                                  # all unit + component + contract tests
pytest -q -m integration                   # needs Postgres + OPA running locally

# 4. lint & type-check — both must pass before PR
ruff check src tests
mypy src

# 5. exercise the CLI scaffold — proves the package built correctly
eaap --help
eaap probe                                 # runs SDK health probes against the bound bindings
tipIf you see asyncio_mode collection errors, you're missing pytest-asyncio. Install with pip install -e ".[dev]"; the bare package install doesn't include it.

CH 04

Cross-cutting concerns.

Datasheet for each concern. Skim the headers; drop into a card when you need detail. Each card lists the files, the failure modes you'll see in production, and the test that catches regressions.

04 · 01

Authentication & Authorisation (JWT + OPA)

security/
Files security/jwt.py · security/opa.py · security/fastapi_dep.py
Tests tests/unit/security/ · tests/integration/security/

Bearer tokens are extracted by OPAAuthorization.requires(), decoded by the bound JWTVerifier, then submitted to OPA at opa_decision_path. Default is fail-closed: any policy or transport error denies. NoOpPolicyEvaluator is dev-only.

Design decisions

  • OPA is the policy decision point — never inline rules
  • fail_closed=True is the default in production
  • UnverifiedJWTDecoder only when an upstream gateway already validated the signature

Failure modes

  • OPA unreachable → PolicyDenialError cascades for all tools
  • Misconfigured opa_url → silent dev-mode fallback if fail_closed=False
  • JWT clock-skew → InvalidTokenError on every request

Oncall note

If 100% of tool_call events show decision_allowed=False with error_code="opa.unreachable", it's an OPA outage, not a policy issue. Check EAAP_SECURITY__OPA_URL reachability first.

Smoke tests

  • pytest tests/unit/security/test_opa.py -q
  • pytest -m integration tests/integration/security/
04 · 02

Audit pipeline (IAuditSink + redactors)

audit/
Files audit/interface.py · audit/buffered.py · audit/sentry.py · audit/datadog.py
Tests tests/unit/audit/ · tests/component/audit/

Every @tool invocation emits an AuditRecord through the audit pipeline: redactor chain → buffered sink → fan-out (file, Sentry, Datadog). Sinks never raise — backend errors are logged and swallowed so audit failures don't break the agent.

Design decisions

  • Redactors run before any sink; PII never reaches a backend
  • Buffered sink writes JSONL via asyncio.to_thread; flush on container stop
  • Optional sinks (Sentry, Datadog) gate imports with ConfigurationError on missing extras

Failure modes

  • Disk full → buffered sink errors logged; events lost (alert on this)
  • Sentry/Datadog rate limit → silent retry by their SDKs; eventual drop
  • New PII pattern un-redacted → audit a sample event in dashboards

Oncall note

If audit events stop arriving, check disk space first, then the bound sinks list (EAAP_AUDIT__SINKS). The agent is not affected — only the audit trail is.

Smoke tests

  • pytest tests/unit/audit/ -q
  • InMemoryAuditSink for unit tests
04 · 03

Observability (OTel + LangFuse + structlog)

observability/
Files observability/real.py · observability/logging.py
Tests tests/unit/observability/ · tests/component/observability/

Three layers: OTel spans for trace topology, LangFuse generations for LLM-specific data (prompt/completion/tokens/cost), structlog with task-isolated ContextVar for log correlation. fail_open is the default — backend errors are logged and swallowed so observability never breaks the agent.

Design decisions

  • One trace_id joins logs, spans, and audit records
  • bind_context() + OTel baggage propagated automatically through asyncio tasks
  • LangFuse trace per task via contextvars.ContextVar

Failure modes

  • OTel collector down → spans buffered then dropped (with warning logs)
  • LangFuse credentials wrong → traces created locally, never delivered
  • Forgetting reset_context() in a finally block → context leaks across requests

Oncall note

If incidents are hard to correlate, check that bind_context() is called early and tenant_id/request_id are bound. Missing binds, not missing infra, is the usual cause.

Smoke tests

  • FakeObservabilityProvider for unit tests
  • pytest tests/component/observability/ -q
04 · 04

Budget enforcement (BudgetService)

budget/ · llm/
Files budget/service.py · llm/litellm_client.py (call sites)
Tests tests/unit/budget/ · tests/integration/budget/

Per-tenant and per-(tenant, agent) caps on tokens and USD. Reset at midnight UTC. Every llm.complete() call is bracketed by check + record. Override format: {"tenant:agent_id": {"daily_usd_limit": N}}.

Design decisions

  • Per-(tenant, agent) overrides take precedence over tenant defaults
  • Reset is idempotent — re-running on the same day doesn't double-charge
  • Cost extracted from LiteLLM's cost lookup tables

Failure modes

  • Misspelled agent_id in overrides → silently ignored, tenant default applies
  • LiteLLM cost lookup missing for a model → cost recorded as 0 (token cap still applies)
  • DB unavailable → defaults to deny (configurable)

Oncall note

Spike in BudgetExceededError? Check the override JSON. A misformatted override silently invalidates the whole override map and falls back to tenant defaults — which may be too tight.

Smoke tests

  • pytest tests/unit/budget/ -q
  • Integration: pytest -m integration tests/integration/budget/
04 · 05

Retry & timeout (internal · tenacity)

llm/ · http
Files llm/litellm_client.py · http clients
Tests tests/unit/llm/test_retry.py

Tenacity-driven, status-aware retries with jitter. Internal-only — agent code never configures retries. Timeouts honour cancellation (asyncio.wait_for). Retry exhaustion raises LLMInvocationError with the original cause attached.

Design decisions

  • Retry only on retryable HTTP statuses (429, 5xx) and connection errors
  • Maximum 3 attempts by default
  • Per-call timeout from EAAP_LLM__REQUEST_TIMEOUT_SECONDS

Failure modes

  • Retries hide rate-limiting → watch for retry_count > 0 as a leading indicator
  • Sticky 5xx from a vendor → all attempts fail; agent gets LLMInvocationError
  • Cancellation during retry doesn't always abort instantly

Oncall note

If LLM p99 latency is high but error rate is low, retries are masking transient errors. The vendor's status page is the place to look.

Smoke tests

  • pytest tests/unit/llm/test_retry.py -q
04 · 06

Schema versioning (SchemaRegistry)

schema/
Files schema/registry.py
Tests tests/unit/schema/test_registry.py

Every (name, version) pair maps to one Pydantic input/output pair. @registry.validate_tool wraps sync or async handlers; @tool auto-registers with the active registry on agent compile.

Design decisions

  • Versions are positive integers — bump on any breaking change
  • Lookups take a snapshot; safe across coroutines on a single event loop
  • replace=True only on intentional reregistration

Failure modes

  • Two registries in one process → SchemaValidationError on cross-lookup
  • Forgot to bump version on schema change → callers silently get the new shape
  • Pydantic validation on egress fails → bug is in the handler, not the schema

Oncall note

Schema mismatches surface as SchemaValidationError with name, version, and errors in details. Use those fields directly in dashboards.

Smoke tests

  • pytest tests/unit/schema/ -q
04 · 07

Secret management (ISecretManager)

config/secrets.py
Files config/secrets.py · persistence/aws_secrets.py (extra)
Tests tests/unit/config/test_secrets.py

Secrets are opaque references (SecretRef(backend, name, version)) resolved at call time. Never inlined in settings. Never logged. The bound ISecretManager handles backend specifics.

Design decisions

  • EnvSecretManager is the dev default; AWSSecretsManager from the prod module
  • SecretResolutionError exposes backend only — not the secret name
  • resolve_optional() is the polite version

Failure modes

  • Secret not in env → SecretResolutionError on first resolve attempt
  • AWS Secrets Manager IAM denied → looks identical to "missing"; check audit logs
  • Logging a resolved secret in a tool handler — fix this in code review

Oncall note

A failed secret resolve at app startup means AICoreApp.__aenter__ never completes. Check container readiness probes — they fail fast.

Smoke tests

  • pytest tests/unit/config/test_secrets.py -q
  • InMemorySecretManager for unit tests
04 · 08

MCP integration (IMCPConnectionFactory)

mcp/
Files mcp/transports.py · mcp/registry.py · mcp/resolver.py
Tests tests/unit/mcp/ · tests/component/mcp/

Connection pool with idle TTL and health checks. Late-binding-safe tool resolution (factory pattern avoids Python's late-binding closure bug). Tool name collisions across servers raise rather than silently picking one.

Design decisions

  • component_id uniquely identifies a server in audit + obs
  • Health checks happen on first acquire after idle TTL
  • Resolver fails fast on tool-name collisions

Failure modes

  • MCP server crashes → next acquire reconnects; in-flight calls fail with MCPTransportError
  • Tool collision across servers → RegistryError at agent compile time
  • Stdio server exits without sending a response → connection orphaned until idle TTL

Oncall note

MCPTransportError rate jump usually means the upstream server is unhealthy, not the SDK pool. Check the server's logs first.

Smoke tests

  • pytest tests/unit/mcp/ -q
  • FakeMCPFactory for unit tests
04 · 09

Memory & compaction (MemoryManager)

agents/memory.py
Files agents/memory.py
Tests tests/unit/agents/test_memory.py

Token-aware summariser. Compaction wraps in asyncio.wait_for so a slow LLM cannot hang an agent. Preserves Essential Entities verbatim across compactions. Replacement (not append) semantics via RemoveMessage(REMOVE_ALL_MESSAGES).

Design decisions

  • Compaction times out → input state returned unchanged, agent continues
  • Token counter falls back to chars/4 heuristic with a debug log
  • Essential entities collected from settings + state

Failure modes

  • Compaction loop — threshold below post-compaction size; tune memory_compaction_target_tokens
  • Lost essential entity — key not in essential_entity_keys and not already in state
  • Tokenizer model mismatch — check token_counter.fallback debug logs

Oncall note

compaction.skipped.budget_exceeded warnings repeated for the same agent → the summariser model is too slow for the configured timeout. Investigate before raising the timeout.

Smoke tests

  • pytest tests/unit/agents/test_memory.py -q
04 · 10

DI & test seam (Container · FakeModule)

di/ · testing/
Files di/container.py · di/module.py · testing/fakes.py
Tests tests/unit/di/ · tests/contract/

Wraps injector. Container.build([modules]) is the entry point; container.override(I, impl) is the test seam. Lifecycle hooks fire on start() and stop() in LIFO order.

Design decisions

  • Singleton scope by default; transient bindings are explicit
  • Overrides don't propagate across forked containers
  • Lifecycle hooks idempotent on repeated start

Failure modes

  • Forgotten override → test silently uses production binding (and may hit real services)
  • Module ordering matters — last bound wins
  • Async teardown can race if hooks aren't awaited

Oncall note

If a test failure says "PostgreSQL connection refused" on a unit test, someone forgot a fake. The contract suite enforces this for new interfaces.

Smoke tests

  • pytest tests/unit/di/ -q
  • pytest tests/contract/ -q
04 · 11

Health & probes (IHealthProbe · HealthSnapshot)

health/
Files health/interface.py · health/probes.py
Tests tests/unit/health/

Probes return ProbeResult(status, details); never raise. AICoreApp.health() runs every bound probe in parallel and aggregates into HealthSnapshot(status, components). Wire to /health for readiness.

Design decisions

  • Probes are quick; budget < 1s each
  • Snapshot is the AND of all probes (any down → down)
  • Probes catch their own exceptions and report degraded

Failure modes

  • Slow probe blocks readiness — set strict timeouts
  • Reading external services in a probe — keep them lightweight
  • Forgetting to bind a probe — silent gap

Oncall note

Liveness vs readiness: probes feed readiness. If liveness fails, the process is wedged — not the dependencies.

Smoke tests

  • pytest tests/unit/health/ -q
  • eaap probe from CLI
CH 05

Development workflow.

Branch, code, test, lint, type-check, PR, merge, release. The same loop, every time.

  1. Branch from main. Name: feat/..., fix/..., refactor/..., docs/..., or chore/....
  2. Write a contract test first if you're touching a public interface. Surface tests in tests/contract/ guard __all__ and the exception hierarchy.
  3. Implement, then run pytest -q. If you touched async code or DI, also run pytest -m component.
  4. Lint: ruff check src tests. Type-check: mypy src (strict mode). Both must pass.
  5. Update CHANGELOG.md if you changed public surface, settings, or behaviour. The release flow expects it.
  6. PR. Squash-merge after review. Use Conventional Commits in titles: feat(scope): ....
  7. Release: tag vX.Y.Z on main; CI builds wheels and uploads to the internal index.
deprecation policyPublic surface deprecations get one full release cycle of warning before removal — emit a DeprecationWarning, document in CHANGELOG, then remove the next minor.

CH 07

Oncall & incident response.

The five most common alerts you'll see and what to do. Each card: symptoms → check → resolve.

SEV · OPA UNREACHABLE

All policy decisions denying

Symptoms: 100% of tool_call events show decision_allowed=False with error_code="opa.unreachable". Agent latency stays normal — they fail fast.

curl -s $EAAP_SECURITY__OPA_URL/health || echo "OPA DOWN"

Resolve: Page the team that operates OPA. Do not flip FAIL_CLOSED=false — that turns enforcement off entirely. Wait it out or roll back the bad OPA bundle.

SEV · LLM RATE LIMITED

LLM calls failing in clusters

Symptoms: LLMInvocationError rate spike, retry_count at the cap, vendor status page shows incident. Latency p99 elevated even when calls succeed (retries running).

grep "LLMInvocationError" logs | head -50 # confirm vendor + status

Resolve: If a single tenant is misbehaving, lower their DAILY_TOKEN_LIMIT. If vendor-wide, switch the affected agents to a different model via EAAP_LLM__DEFAULT_MODEL and let LiteLLM route to a backup provider.

SEV · POSTGRES UNREACHABLE

Checkpoints + budget enforcement broken

Symptoms: StorageError on agent start. Health probe red. Stateful agents fail; stateless tools may still work.

psql "$EAAP_DATABASE__URL" -c "SELECT 1"

Resolve: Page the DB team. Until back, scale stateless workloads only. The SDK's checkpoint saver is fail-loud — agents won't silently lose state.

SEV · MCP SERVER UNHEALTHY

Tools from one MCP server time out

Symptoms: MCPTransportError rate jump for one component_id. Other tools unaffected.

curl http://<mcp-server>/health # most MCP servers expose this

Resolve: Restart the MCP server. The pool reconnects on next acquire. If recurring, raise EAAP_MCP__IDLE_TTL_SECONDS or check the server's logs for crashes.

SEV · AUDIT SINK FAILING

No audit events arriving

Symptoms: Audit dashboard shows no events for >5 minutes. Agents working fine. Background log line: audit.<sink>.record_failed.

df -h $EAAP_AUDIT__BUFFER_DIR # disk full?

Resolve: If buffered file sink: free disk. If Sentry/Datadog: check vendor status. Audit pipeline fails silently by design — agents don't break — but compliance is now accruing debt; treat as P2.

postmortem ruleEvery Sev1 / Sev2 produces a doc with: timeline, root cause, what we'll change. The doc lives in docs/incidents/YYYY-MM-DD-short-name.md. Add a regression test pointing at the failure mode.
CH 08

Platform cookbook.

Common platform-engineering tasks and the smallest sequence that gets them done.

→ Add a new I* interface

  1. Define the abstract base in src/ai_core/di/interfaces.py (or a more specific module). Document return types and exception contracts in the docstring.
  2. Implement the production binding alongside (or in its subpackage). Wire it in the relevant DI module (AgentModule / ProductionSecurityModule).
  3. Add a fake in src/ai_core/testing/fakes.py and bind it in FakeModule.
  4. Add a contract test in tests/contract/ verifying both prod and fake satisfy the interface.
  5. Update the table in CH 03 of this guide. (Yes, really.)
  6. Document in CHANGELOG under "Added".

→ Add a new audit sink

  1. Implement IAuditSink in src/ai_core/audit/<name>.py. Wrap optional imports with ImportError → ConfigurationError(error_code=ErrorCode.CONFIG_OPTIONAL_DEP_MISSING, details={"extra": "<name>"}).
  2. Add the optional dep to pyproject.toml under [project.optional-dependencies].
  3. Wire it into the audit fan-out in AgentModule when EAAP_AUDIT__SINKS contains its name.
  4. record must never raise — catch Exception, log a warning, return.
  5. Add a unit test that asserts the never-raise contract under a forced backend exception.

→ Add a new settings group

  1. Define a Pydantic BaseSettings subclass in src/ai_core/config/settings.py.
  2. Use EAAP_ prefix and __ nested separator (e.g. EAAP_LLM__DEFAULT_MODEL).
  3. Attach to AppSettings as a field with a default factory.
  4. Regenerate docs/settings.md via scripts/regen-settings-docs.py.
  5. Add an entry to the survival-guide CH 06 env-var block.

→ Migrate a dependency major version (e.g. LangGraph 1.0)

  1. Stage on a feature branch. Bump the dep range in pyproject.toml; run the test matrix.
  2. Identify breakage. Triage by location: internal-only changes are easy; public-surface changes need a deprecation cycle.
  3. Migrate internals; keep the public API stable. The whole point of the SDK is that agents don't have to migrate.
  4. Add regression tests for any behaviour you discovered while migrating.
  5. CHANGELOG entry describing what changed for agent authors (usually "nothing").
  6. Release as a minor (not patch) to signal the dep change.

→ Add a new health probe

  1. Implement IHealthProbe in src/ai_core/health/probes.py. Must complete in <1s.
  2. Catch every exception inside probe(); return ProbeResult(status="degraded", details={...}) on error. Never raise.
  3. Bind it in AgentModule.configure.
  4. Add a unit test that forces the underlying call to throw and asserts status != "up".
  5. Verify eaap probe shows it.
CH 09

Anti-patterns.

Things that look reasonable in week one and burn you in week six. Block these in code review.

Reading os.environ in SDK code

Settings are typed and validated. Bypassing them means dev/prod parity breaks invisibly.

Add to AppSettings; reference via the container.
Catching Exception broadly

Hides BudgetExceededError, PolicyDenialError, and SDK contract errors.

Catch the specific SDK exception. EAAPBaseException if you need a category.
Breaking the public surface without deprecation

The contract test catches removals; the surprise lands on agent teams. Don't.

Emit DeprecationWarning, ship a release, then remove next minor.
Adding a dep without an optional-extra guard

Forces every consumer to install your favourite cloud SDK. They won't thank you.

Use [project.optional-dependencies] + ImportError → ConfigurationError.
Skipping the contract tests

The contract suite is the social agreement with the business AI team. Bypassing it breaks trust.

Run pytest tests/contract/ -q as part of your normal flow. CI runs it too.
Hand-rolling DI to "make a test simpler"

Test fixtures that build their own container without using FakeModule drift from the real binding pattern.

Use Container.build([AgentModule(), FakeModule()]) + container.override().
Logging resolved secrets

The redactor chain catches some patterns; defence-in-depth means avoiding the mistake at the source.

Log the SecretRef, never the resolved value.
Putting business logic in the SDK

The SDK serves every agent. Anything specific to one team's domain belongs in their package.

Generic primitives only. Opinion belongs in the agent code.
CH 10

Working with the AI team.

The platform/business contract works only if the relationship works. A short etiquette guide.

Office hours

Hold a 30-minute weekly slot for the business AI team to surface friction. Treat the friction list as input to the next sprint, not as a complaint dump.

Feature requests

If two agent teams independently ask for the same thing, it's a missing SDK feature. Build it once. If only one team asks, hold the bar — they may be the wrong abstraction looking for help.

Breaking changes

Email + Slack + CHANGELOG. Announce one release ahead. Pair-debug with the affected team on the day of upgrade. Never silent.

Don't take their work

Tempting to "just write the prompt for them" when an agent isn't behaving. Don't — it dissolves the role boundary and means you'll be on the hook for prompt quality next quarter.
