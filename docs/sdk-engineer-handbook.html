How to read this

Each capability card follows the same shape: USE WHEN tells you the case the capability solves; DON'T USE WHEN warns you off; BEHIND THE SCENES says what the SDK does for you so you don't have to; GOTCHAS covers the things people learn the hard way; TEST SEAM tells you which fake to inject in unit tests.

The code samples are the minimum that works. Production usage is the same code with more configuration, not different code. If you can't see the API you need, look at src/ai_core/__init__.py — anything in __all__ is supported public surface.

CH 01

Bootstrap.

Install the SDK, scaffold a project, and run a working agent in under five minutes. Skip the rest of the chapter once your "hello, agent" round-trips. 

$ shellinstall + scaffold# 1.  Install
pip install ai-eaap-sdk

# 2.  Scaffold a new project (uses the bundled CLI)
eaap new my-agents
cd my-agents

# 3.  Run the example agent against a local OPA + dev settings
python -m my_agents.main
my_agents/main.py~30 lines · the whole hello-worldfrom ai_core import AICoreApp, BaseAgent, tool
from pydantic import BaseModel

class HiIn(BaseModel):  name: str
class HiOut(BaseModel): greeting: str

@tool(name="say_hi", version=1)
async def say_hi(p: HiIn) -> HiOut:
    return HiOut(greeting=f"Hi {p.name}!")

class Greeter(BaseAgent):
    agent_id = "greeter"
    def system_prompt(self) -> str:
        return "You greet people warmly and end with a smiley."
    def tools(self):
        return [say_hi]

async def main() -> None:
    async with AICoreApp() as app:
        agent = app.agent(Greeter)
        result = await agent.invoke({"messages": [
            {"role": "user", "content": "hello!"}
        ]})
        print(result["messages"][-1])

if __name__ == "__main__":
    import asyncio; asyncio.run(main())

What just happened
AICoreApp() built a DI container, bound development defaults (no-op policy, env-based secrets, in-memory checkpoints, stub observability), constructed the agent, and routed your message through schema validation → policy → handler → audit → observability. You wrote ~30 lines; the SDK supplied the other ~10,000 lines you didn't have to.

Twelve datasheet cards, one per primitive. The SDK is bigger than this — these are the pieces you'll touch every week.

02 · 01

@tool · the tool decorator

ai_core.tools.tool

Use when

  • You want the agent's LLM to be able to call a function with typed inputs and outputs.
  • You want OPA policy evaluated before the function runs.
  • You want the call audited automatically.

Don't use when

  • You're calling something internally without LLM mediation — that's a regular function.
  • The function is sync. Tools must be async.
  • The function is a method or classmethod — tools are module-level only.
tools/lookup_ticket.pycanonical shapefrom ai_core.tools import tool
from pydantic import BaseModel, Field

class LookupTicketIn(BaseModel):
    ticket_id: str = Field(..., min_length=1, pattern=r"^T-\d+$")

class LookupTicketOut(BaseModel):
    ticket_id: str
    status:    str
    summary:   str

@tool(
    name="lookup_ticket",
    version=1,
    description="Fetch a ticket by id.",
    opa_path="eaap/support/lookup",    # or None to skip policy
)
async def lookup_ticket(p: LookupTicketIn) -> LookupTicketOut:
    row = await tickets.fetch(p.ticket_id)
    return LookupTicketOut(**row)

Behind the scenes

Schema is registered under (name, version). Inputs are parsed and validated; outputs are validated. OPA is consulted via the bound IPolicyEvaluator. Calls emit an AuditRecord through the audit pipeline.

Gotchas

opa_path defaults to "eaap/agent/tool_call/allow". Set it explicitly for prod tools so policies are scoped. Pass None only for read-only/system tools, with intent.

Test seam

Bind FakePolicyEvaluator(default_allow=True) in your test container; assert your tool was called with the right parsed payload. No HTTP, no OPA.

02 · 02

BaseAgent · the agent base class

ai_core.agents.BaseAgent

Use when

  • You're building an agent — anything that loops over an LLM with tools.
  • You want compaction, observability, tool dispatch, and MCP wiring "for free".

Don't use when

  • You're writing a one-shot LLM call — just use the ILLMClient directly.
  • You need a fundamentally different graph topology and don't want any of the scaffolding.
agents/support_agent.pyoverride 1–3 methodsfrom ai_core import BaseAgent
from ai_core.mcp import MCPServerSpec
from .tools.lookup_ticket import lookup_ticket
from .tools.create_ticket import create_ticket

class SupportAgent(BaseAgent):
    agent_id = "customer-support"          # used for budget + tracing

    def system_prompt(self) -> str:
        return "You help customers resolve support tickets..."

    def tools(self):
        return [lookup_ticket, create_ticket]

    def mcp_servers(self):                  # optional
        return [MCPServerSpec(
            component_id="kb-mcp",
            transport="stdio",
            command=["python", "-m", "kb_mcp.server"],
        )]

Behind the scenes

Compiled to a LangGraph with a compaction node and an agent node. Tools and MCP-resolved tools are merged into one tool set advertised to the LLM. Compaction wraps in asyncio.wait_for so a slow LLM can't hang you.

Gotchas

agent_id drives budget keys and trace attributes — pick a stable string and don't change it lightly. Override extend_graph for custom nodes; don't subclass compile().

Test seam

Override the LLM binding with a stub that returns canned responses; assert the tool dispatch path. FakeObservabilityProvider lets you assert spans without a collector.

02 · 03

ILLMClient · LLM calls

ai_core.di.interfaces.ILLMClient

Use when

  • You need a one-shot completion outside an agent loop.
  • You're embedding an LLM call inside a tool handler and want the SDK's retries, timeouts, token counting, and budget enforcement.

Don't use when

  • You're inside an agent — the agent already owns the LLM client. Don't construct your own.
  • You want to call OpenAI/Anthropic directly. The SDK routes everything through LiteLLM; do that one too.
tools/summarise.pyresolve from containerfrom ai_core.di.interfaces import ILLMClient
from ai_core.tools import tool
from injector import inject

class SummariseIn(BaseModel):  text: str; max_tokens: int = 200
class SummariseOut(BaseModel): summary: str

@tool(name="summarise", version=1)
async def summarise(p: SummariseIn) -> SummariseOut:
    llm: ILLMClient = current_container().get(ILLMClient)
    resp = await llm.complete(
        model="gpt-4o-mini",         # or read from settings
        messages=[
            {"role": "system", "content": "Summarise concisely."},
            {"role": "user",   "content": p.text},
        ],
        max_tokens=p.max_tokens,
        tenant_id=current_tenant(),
        agent_id=current_agent(),
        temperature=0.0,
    )
    return SummariseOut(summary=resp.content.strip())

Behind the scenes

LiteLLM under the hood. Tenacity-driven retries with jitter; timeouts honour cancellation. Token usage and cost are extracted into an LLMUsage attached to the response. The budget service rejects calls past the daily limit.

Gotchas

Always pass tenant_id and agent_id — without them, budget enforcement and observability lose their join keys.

Test seam

Bind a FakeLLMClient that returns canned LLMResponses with the usage you want to test. No network calls.

02 · 04

MemoryManager · history compaction

ai_core.agents.memory

Use when

  • Your agent has long conversations that exceed the model's context.
  • You want a budget-aware summariser to compress history while preserving Essential Entities.

Don't use when

  • Your agent is single-turn — there's nothing to compact.
  • You need a fundamentally different memory strategy (recency only, semantic clustering) — subclass IMemoryManager and bind your own.
.env / settingsconfiguration only# Threshold above which compaction runs (in tokens)
EAAP_AGENT__MEMORY_COMPACTION_TOKEN_THRESHOLD=12000

# Target post-compaction size — passed into the summariser prompt
EAAP_AGENT__MEMORY_COMPACTION_TARGET_TOKENS=2000

# Compaction is wrapped in asyncio.wait_for(); slow LLM can't hang the agent
EAAP_AGENT__COMPACTION_TIMEOUT_SECONDS=30

# Keys whose values are preserved verbatim across compactions
EAAP_AGENT__ESSENTIAL_ENTITY_KEYS=customer_id,ticket_id,policy_number

Behind the scenes

The compaction node runs a one-shot summariser with a system prompt that demands verbatim preservation of Essential Entities. Output replaces older messages via the add_messages reducer.

Gotchas

If compaction times out, the agent run continues with the unmodified state — it does not crash. Watch for compaction.skipped.budget_exceeded in logs.

Test seam

Inject a fake TokenCounter with deterministic counts and a fake LLM that returns a known summary. Assert the resulting state shape.

02 · 05

BudgetService · cost & token limits

ai_core.di.interfaces.IBudgetService

Use when

  • You need to cap inference spend per tenant or per agent.
  • You want a "fail loud" mechanism before an LLM bill becomes a problem.

Don't use when

  • You think you'll do it later. By the time you do, you've already burned through the budget once.
.env / settingstenant + agent limits# Default per-tenant/day cap
EAAP_AGENT__DAILY_TOKEN_LIMIT=2000000
EAAP_AGENT__DAILY_USD_LIMIT=200.00

# Per-(tenant, agent) overrides — finer grain than tenant alone
EAAP_AGENT__BUDGET_OVERRIDES='{"acme:billing-agent": {"daily_usd_limit": 50.00}}'
handling BudgetExceededErrorgraceful denialfrom ai_core import BudgetExceededError

try:
    resp = await llm.complete(model=..., messages=..., tenant_id=...)
except BudgetExceededError as e:
    # Don't retry — return a friendly error to the user.
    return {"error": "Daily budget exceeded for tenant",
            "reason": e.message}

Behind the scenes

Each llm.complete() call is bracketed by a budget check + usage recording. State resets at midnight UTC. Per-(tenant, agent) overrides take precedence over tenant defaults.

Gotchas

Override format is JSON. The key is "{tenant_id}:{agent_id}". Misspell the agent_id and the override silently doesn't apply.

Test seam

FakeBudgetService with pre-loaded balances. Assert your tool returns a graceful error, not a 500, on exhaustion.

02 · 06

bind_context() · observability context

ai_core.observability.logging.bind_context

Use when

  • You want tenant_id, user_id, request_id, etc. on every log line, span, and audit event for the rest of the request.

Don't use when

  • You're inside an agent invocation — the SDK already binds agent_id and the trace id.
  • You want to set values that should be private — context is exported to OTel baggage.
api/handlers.pyFastAPI handlerfrom ai_core.observability.logging import bind_context, reset_context

@app.post("/run")
async def run(req: RunRequest, principal: AuthorizedPrincipal = Depends(authz.requires(action="agent.run"))):
    tokens = bind_context(
        tenant_id=principal.claims["tenant_id"],
        user_id=principal.subject,
        request_id=req.request_id,
    )
    try:
        return await support_agent.invoke(req.payload)
    finally:
        reset_context(tokens)        # cleanup so values don't leak into the next request

Behind the scenes

Values land in a structlog ContextVar AND OTel baggage simultaneously, so they appear on log records and span attributes both. ContextVar is task-isolated — each request keeps its own values.

Gotchas

Always pair bind_context() with reset_context() in a finally block. Spawned tasks (asyncio.create_task) inherit the bound context — that's usually what you want, but keep it in mind for fan-out.

Test seam

Bind FakeObservabilityProvider and inspect provider.recorded_events after the call. No collector required.

02 · 07

SchemaRegistry · versioned contracts

ai_core.schema.SchemaRegistry

Use when

  • You need a versioned input/output contract for messages crossing tools, agents, or MCP boundaries.
  • You want one place to change a schema and have every caller re-validate.

Don't use when

  • The data lives entirely inside one function. Pydantic alone is enough.
contracts/create_ticket.pyregister + decoratefrom pydantic import BaseModel
from ai_core.schema import SchemaRegistry

registry = SchemaRegistry()

class CreateTicketIn(BaseModel):
    title: str; priority: str = "low"

class CreateTicketOut(BaseModel):
    ticket_id: str

registry.register("create_ticket", 1,
    input_schema=CreateTicketIn,
    output_schema=CreateTicketOut,
    description="Open a new support ticket.")

@registry.validate_tool("create_ticket", version=1)
async def create_ticket(p: CreateTicketIn) -> CreateTicketOut:
    ticket = await backend.create(p.title, p.priority)
    return CreateTicketOut(ticket_id=ticket.id)

Behind the scenes

The decorator parses the input model on call, validates the return against the output model, and raises SchemaValidationError with structured details on either side. Latest version is resolved on lookup if you don't pin one.

Gotchas

Bumping version creates a new entry; it does not replace the old one. Pass replace=True only if you really mean it.

Test seam

Use a per-test registry instance to avoid cross-test pollution. Pydantic models test fine on their own.

02 · 08

SecretRef · opaque secret references

ai_core.config.secrets

Use when

  • You need an API key, DB password, or token at runtime.
  • You want the same code to work in dev (env var) and prod (Vault / AWS SM) without changes.

Don't use when

  • You're tempted to hardcode it. Don't.
  • You're tempted to put it in source-controlled config. Don't.
tools/post_to_slack.pyresolve at call timefrom ai_core.config.secrets import SecretRef, ISecretManager

@tool(name="post_to_slack", version=1, opa_path="eaap/notify/slack")
async def post_to_slack(p: SlackPostIn) -> SlackPostOut:
    secrets: ISecretManager = current_container().get(ISecretManager)
    webhook = await secrets.resolve(SecretRef(
        backend="env",            # or "aws-sm", "vault", ...
        name="SLACK_WEBHOOK_URL",
    ))
    await http.post(webhook, json={"text": p.message})
    return SlackPostOut(ok=True)

Behind the scenes

The bound ISecretManager handles the backend specifics. EnvSecretManager is the dev default; AWSSecretsManager is bound by the prod module.

Gotchas

Don't log a resolved secret. Don't put resolved values in exception messages. The SDK redactor chain catches common mistakes; defence-in-depth means avoiding the mistake in the first place.

Test seam

Bind InMemorySecretManager({"SLACK_WEBHOOK_URL": "https://example/"}). No env mutation, no real backend.

02 · 09

MCPServerSpec · MCP server integration

ai_core.mcp

Use when

  • You want to surface tools from an MCP server (your own or a vendor's) inside an agent.
  • You want connection pooling, lifecycle, and health checks handled for you.

Don't use when

  • The "MCP server" is really a regular HTTP API. Wrap it as a @tool instead.
agents/research_agent.pydeclare server, agent picks up toolsfrom ai_core.mcp import MCPServerSpec
from ai_core import BaseAgent

class ResearchAgent(BaseAgent):
    agent_id = "research"
    def system_prompt(self) -> str:
        return "You research and cite sources."
    def mcp_servers(self):
        return [
            MCPServerSpec(
                component_id="web-search",
                transport="http",
                url="http://web-search.internal:8080",
            ),
            MCPServerSpec(
                component_id="kb",
                transport="stdio",
                command=["python", "-m", "kb_mcp.server"],
            ),
        ]

Behind the scenes

Connection pool with idle TTL and health checks. Tools surfaced by the server are merged with the agent's tools(). Resources and prompts are available via list_prompts() / get_prompt() helpers on the agent.

Gotchas

If two MCP servers expose a tool with the same name, the resolver raises rather than silently picking one. Either rename the tool upstream or scope it via component_id.

Test seam

Bind a fake IMCPConnectionFactory that returns a stub session with the tool list you want to test against.

02 · 10

AuditRecord · structured audit events

ai_core.audit

Use when

  • You don't have to do anything — every @tool call already emits an AuditRecord through the audit pipeline.
  • You're emitting an audit event from non-tool code (custom workflow step, batch job).

Don't use when

  • You want to log debug information. Use structlog, not audit.
  • You're tempted to drop PII into payload. The redactor chain may catch it; better not to put it there.
workflows/risk_scoring.pyemit a custom audit eventfrom ai_core import AuditRecord, AuditEvent, IAuditSink

audit: IAuditSink = current_container().get(IAuditSink)

await audit.record(AuditRecord(
    event=AuditEvent.DECISION,
    tool_name="risk_score",
    tool_version=2,
    agent_id="risk-agent",
    tenant_id=tenant,
    decision_path="eaap/risk/auto_decline",
    decision_allowed=False,
    decision_reason="score below threshold",
    payload={"score": 0.34},      # goes through redactor chain
    latency_ms=120,
))

Behind the scenes

Records run through a redactor chain (PII patterns) and fan out to the bound sinks: BufferedAuditSink (JSONL files), SentryAuditSink, DatadogAuditSink. Sinks never raise; backend errors are logged and swallowed.

Gotchas

Audit is not a replacement for logs. Audit is the durable, queryable trail of decisions. Logs are the operational stream. Don't conflate them.

Test seam

Bind InMemoryAuditSink; assert sink.records[-1].decision_allowed is False. No filesystem or HTTP.

02 · 11

Container.override() · DI test seam

ai_core.di

Use when

  • You're writing a unit test and want a fake for one (or all) dependencies.
  • You want to swap one production binding (e.g. policy evaluator) without forking modules.

Don't use when

  • You're tempted to monkey-patch. The DI container is the only test seam — please use it.
  • Production code wants to override at runtime. That's a configuration problem, not a DI problem.
tests/test_lookup_ticket.pyoverride + assertfrom ai_core.di import Container
from ai_core.di.module import AgentModule
from ai_core.testing import FakePolicyEvaluator, FakeBudgetService, FakeObservabilityProvider, FakeModule
from ai_core.di.interfaces import IPolicyEvaluator
import pytest

@pytest.fixture
def container():
    c = Container.build([AgentModule(), FakeModule()])
    c.override(IPolicyEvaluator, FakePolicyEvaluator(default_allow=True))
    return c

async def test_lookup_ticket_allowed(container):
    invoker = container.get(ToolInvoker)
    out = await invoker.invoke(lookup_ticket,
        arguments={"ticket_id": "T-42"},
        agent_id="customer-support", tenant_id="acme")
    assert out.status == "open"

Behind the scenes

Container.override(I, impl) replaces the binding at the test scope. Lifecycle hooks for fakes still fire (start/stop). The real container code is exercised — only the dependencies change.

Gotchas

Overrides don't propagate through forked containers. Build the container per test (or per module) and tear it down.

02 · 12

OPAAuthorization · FastAPI dependency

ai_core.security.fastapi_dep

Use when

  • You're exposing your agent over HTTP via FastAPI and want JWT + OPA on every endpoint.

Don't use when

  • You're inside an internal worker — there's no HTTP surface, so no JWT, so no need for this.
api/app.pyone dependency, every routefrom fastapi import FastAPI, Depends
from ai_core.di import Container
from ai_core.di.module import AgentModule, ProductionSecurityModule
from ai_core.security import OPAAuthorization, AuthorizedPrincipal

container = Container.build([AgentModule(), ProductionSecurityModule()])
authz = OPAAuthorization(container, decision_path="eaap/api/allow")
app = FastAPI()

@app.post("/agents/support/run")
async def run_support(
    req: RunRequest,
    principal: AuthorizedPrincipal = Depends(
        authz.requires(action="agent.run", resource="customer-support")
    ),
):
    return await support_agent.invoke(req.payload, tenant_id=principal.claims["tenant_id"])

Behind the scenes

Bearer token extracted, JWT verified via the bound JWTVerifier, OPA consulted with {user, action, resource, claims, request}, 401/403 raised on rejection. AuthorizedPrincipal exposes verified claims and OPA obligations to the handler.

Gotchas

AgentModule alone binds a no-op policy evaluator — fine for dev, NOT fine for prod. ProductionSecurityModule is required for actual enforcement.

Test seam

Override IPolicyEvaluator with FakePolicyEvaluator and use FastAPI's TestClient. JWTs are decoded but not verified in the dev verifier.

Anatomy of a tool call.

The capability cards above plug into a single pipeline. When you call a tool through the SDK, it passes through these stages every time. Most failures map to a stage; knowing which one collapses debugging time.

CH 03

Recipes.

Common end-to-end patterns. Each recipe is a sequence of small steps you can copy.

Add a new tool to an existing agent

  1. Create tools/your_tool.py with a Pydantic input/output and an async def handler.
  2. Decorate with @tool(name=..., version=1, opa_path=...).
  3. Add the import to the agent class's tools() return list.
  4. Add a unit test that calls ToolInvoker.invoke with a fake policy evaluator that allows.
  5. Add an OPA rule under your opa_path in the policy bundle.

Switch from GPT-4o-mini to Claude Sonnet

  1. Update EAAP_LLM__DEFAULT_MODEL=anthropic/claude-sonnet-4-5.
  2. Set the credentials secret (ANTHROPIC_API_KEY) and reference it via SecretRef if you're not using env directly.
  3. Re-run your agent. No code changes — LiteLLM handles the protocol differences.
  4. Re-tune temperature / max_tokens if your evals shift.

Write a unit test for an agent without external services

  1. In conftest.py, build a container with AgentModule() + FakeModule().
  2. Override ILLMClient with a stub returning canned LLMResponses.
  3. Override IPolicyEvaluator with FakePolicyEvaluator(default_allow=True).
  4. Construct the agent via container.get(YourAgent) and assert behaviour.
  5. Run with pytest -q — no Postgres, no OPA, no LLM keys required.

Add a per-tenant budget override

  1. Set EAAP_AGENT__DAILY_USD_LIMIT to your tenant default.
  2. Set EAAP_AGENT__BUDGET_OVERRIDES as JSON: {"acme:billing-agent": {"daily_usd_limit": 50.00}}.
  3. Restart the app. The override applies the next time BudgetService.check() runs for that pair.
  4. Add an alerting rule on the budget.exceeded audit event for that tenant.

Wire an MCP server into an agent

  1. Decide on transport — "stdio" for local subprocess, "http" for a network server.
  2. Add an MCPServerSpec to your agent's mcp_servers() with a unique component_id.
  3. Run the agent. The SDK pools connections and surfaces the server's tools to the LLM automatically.
  4. If two servers expose tools with colliding names, rename upstream — the resolver fails-fast on collision rather than silently picking one.

Deploy an agent behind FastAPI with JWT + OPA

  1. Build the container with AgentModule() + ProductionSecurityModule() in app.py.
  2. Construct OPAAuthorization(container, decision_path="eaap/api/allow").
  3. Add Depends(authz.requires(action="agent.run")) to each route.
  4. Wrap your handler in bind_context() / reset_context() with tenant_id, user_id, request_id.
  5. Expose /health backed by the SDK's health probes (already wired by AICoreApp).

CH 05

Anti-patterns.

The shortcuts that look reasonable in week one and burn you in week six.

ANTI · 01
Reading os.environ directly inside an agent or tool

Settings are typed and validated. Reading the env directly bypasses validation and means your agent breaks differently in dev and prod.

Use AppSettings from the container; resolve secrets via SecretRef.

ANTI · 02
Catching Exception around an LLM call to "swallow errors gracefully"

You'll mask BudgetExceededError and PolicyDenialError and look like you're doing something safe. You aren't.

Catch the specific SDK exception and return a structured error to the caller.

ANTI · 03
Sharing one httpx.AsyncClient across agents by hand

You'll leak file descriptors, fight asyncio task isolation, and reinvent the SDK's connection pool poorly.

Let the SDK construct clients via DI; aclose() is wired into container teardown.

ANTI · 04
Monkey-patching for tests

monkeypatch.setattr(...) on SDK internals is fragile and breaks across versions.

Use Container.override(I, fake). Every interface has a fake in ai_core.testing.

ANTI · 05
Logging customer data directly in tool handlers

Audit goes through a redactor chain. Plain log.info(customer) doesn't.

If it should be durable, send it through AuditRecord. If it's debug, sanitise before logging.

ANTI · 06
Skipping tenant_id in llm.complete() "for now"

Without it, budget enforcement silently doesn't apply. The first time you find out is on the AWS bill.

Pass tenant_id on every call, even in tests. FakeBudgetService ignores it cheaply.
